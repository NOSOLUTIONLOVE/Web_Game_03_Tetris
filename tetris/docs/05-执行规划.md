## 二、总体策略

### 2.1 核心策略：分阶段交付

按 8 个阶段（A-H）实施，每个阶段独立可验证：

```
A 脚手架 ──► B 配置+方块 ──► C 核心算法 ──► D 引擎 ──► E 渲染 ──► F 状态 ──► G UI ──► H 验证
  0.5d         0.5d           1d           1d         0.5d        0.3d       0.5d       0.5d
```

### 2.2 阶段依赖关系

```
A（必须最先）
  │
  ├──► B ──► C ──► D ═══► E ═══► F ═══► G ═══► H
       │     │     │       │       │       │       │
       │     │     │       │       │       │       └──► 部署上线
       │     │     │       │       │       └──► 完整 UI
       │     │     │       │       └──► store 桥接
       │     │     │       └──► Canvas 多区域
       │     │     └──► 状态机 + 计分 + Hold
       │     └──► 碰撞 + 锁定 + 消行
       └──► CONFIG + SRS 踢墙表
```

## 三、阶段详解

### 📦 A：脚手架（0.5d）✅

**目标**：项目能 `npm run dev` 跑起来。

#### A.1 初始化 Vite 项目

```bash
cd games/tetris
npm create vite@latest . -- --template react-ts
npm install
```

#### A.2 调整 tsconfig（strict + path alias）

```json
{
  "compilerOptions": {
    "strict": true,
    "paths": { "@/*": ["./src/*"] }
  }
}
```

> **注**：`noUncheckedIndexedAccess` 未启用（网格/矩阵访问频繁，避免 undefined 噪音），详见 [03 - 技术选型](./03-技术选型.md)。

#### A.3 配置 Tailwind + shadcn/ui

```bash
npm install -D tailwindcss postcss autoprefixer
npm install class-variance-authority clsx tailwind-merge lucide-react
# 手动复制 7 个 shadcn 组件（自 2048）
```

#### A.4 验收

- [x] `npm run dev` 启动无错（端口 5175）
- [x] 默认页面显示 TETRIS 标题
- [x] `npm run build` 打包成功

### 🕹️ D：GameEngine 编排（1d）✅

**目标**：完整游戏循环 + 状态机 + 事件回调。

#### D.1 状态机

```
menu ──confirm──► playing ──pause──► paused ──pause──► playing
                       │
                       └─触顶──► over ──confirm──► playing
```

#### D.2 核心功能

- 60Hz 固定步长主循环（LOGIC_STEP = 16.67ms）
- 重力下落（按当前等级 tickRate）
- 软降（1/10 速度）/ 硬降（一键到底）
- 旋转：CW / CCW / 180°（含 SRS 踢墙）
- 7-bag 随机 + Next 队列（4 个 = 3 显示 + 1 缓冲）
- Hold 系统（含 holdUsed 锁）
- NES 经典计分 + 连击
- 等级提升（每 10 行 + 1，速度按 0.85 系数递减）
- Game Over 判定 + 破纪录持久化

#### D.3 单元测试

- 32 个测试覆盖：状态机、计分、等级、消行、Hold、暂停、输入响应、Audio
- happy-dom 环境
- Canvas getContext 手动 stub

#### D.4 验收

- [x] GameEngine.test.ts 32 个测试全绿
- [x] 状态机正确切换

## 四、关键决策记录

| 决策 | 备选 | 选定 | 理由 |
| --- | --- | --- | --- |
| 渲染方式 | Canvas / SVG / WebGL | Canvas 2D | 性能 + 复杂度平衡 |
| 状态管理 | Redux / Zustand / Jotai | Zustand | 1.5KB 极简 |
| 测试环境 | jsdom / happy-dom | happy-dom | 比 jsdom 轻 70% |
| 动画 | CSS / Framer Motion / GSAP | Framer Motion | 项目库统一 |
| Hold 锁实现 | 锁定到游戏结束 / 每方块 1 次 | 每方块 1 次 | Tetris Guideline |
| 7-bag vs 纯随机 | 7-bag / Math.random | 7-bag | Guideline 标准 |
| 速度曲线 | 线性 / 指数 | 0.85 指数衰减 | 经典玩法 |
