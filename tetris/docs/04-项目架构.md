## 一、整体架构

Tetris 采用 **三层分离架构**：

```
┌──────────────────────────────────────────────────────┐
│  UI 层（React + Framer Motion）                      │
│  ├─ TetrisGame (Canvas 挂载 + Engine Context)        │
│  ├─ HUD / MainMenu / PauseOverlay / GameOverModal    │
│  └─ SettingsPanel / Overlays / Footer                │
└──────────────────┬───────────────────────────────────┘
                   │ Engine Context + Zustand
                   ▼
┌──────────────────────────────────────────────────────┐
│  状态层（Zustand + persist）                          │
│  └─ useGameStore: phase / score / highScore / ...    │
└──────────────────┬───────────────────────────────────┘
                   │ callbacks (onPhaseChange, ...)
                   ▼
┌──────────────────────────────────────────────────────┐
│  引擎层（Pure TypeScript，框架无关）                  │
│  ├─ GameEngine（编排 + 状态机 + 主循环）              │
│  ├─ Board（10×20 + 2 行 buffer 网格）                │
│  ├─ Tetromino（7 种方块 + 旋转 + SRS 踢墙）          │
│  ├─ Bag（7-bag 随机生成）                            │
│  ├─ Renderer（多区域 Canvas 2D 渲染）                │
│  ├─ Input（键盘 + 触屏）                             │
│  └─ AudioSystem（Web Audio 合成）                    │
└──────────────────────────────────────────────────────┘
```

### 4.1 引擎 → UI（事件驱动）

```

       ▼

       │
       ▼

       │                       │
       │                       ▼
       │                 handleLineClear(rows)
       │                       │
       │                       ├──► score += base × level + combo
       │                       ├──► lines += count
       │                       ├──► callbacks.onLinesClear(count, isTetris)
       │                       ├──► callbacks.onLevelUp(newLevel)
       │                       └──► board.clearLines(rows)
       │
       ▼

       │
       ▼
isGameOver(current) ──► gameOver() ──► callbacks.onPhaseChange('over')
       │
       ▼
pushState() ──► callbacks.onStateChange(snapshot)
       │
       ▼

       │
       ▼

```

  ▼

## 五、关键设计决策

### 5.1 三层分离

| 层 | 职责 | 依赖 |
| --- | --- | --- |
| UI（React） | 渲染 + 事件绑定 | Engine Context + Zustand |
| State（Zustand） | 全局可观察状态 | — |
| Engine（Pure TS） | 游戏逻辑 + 渲染 | 仅 DOM API（Canvas / Audio / Keyboard） |

**好处**：
- 引擎可独立单元测试（不依赖 React）
- UI 可独立替换（如未来加 React Native）
- 状态层 1.5KB，无样板代码

### 5.2 引擎不订阅 store

**问题**：60fps 主循环若每帧更新 Zustand → React re-render，会卡顿。

**解决**：
- 引擎内部状态私有（`this.score`、`this.lines` 等）
- 通过 **callbacks** 推送事件（消行 / 升级 / 状态切换）
- 仅关键事件触发 store 更新，避免每帧更新

**事件触发时机**：
| 事件 | 时机 |
| --- | --- |
| `onPhaseChange` | 状态切换 |
| `onStateChange` | 移动 / 旋转 / 锁定后 |
| `onLinesClear` | 消行 |
| `onLevelUp` | 升级 |
| `onGameOver` | 死亡 |

### 5.3 Canvas 一次性绘制

**策略**：每帧 `clear()` + 全量重绘（不增量更新）。

**原因**：
- Tetris 元素少（最多 200 格子 + 当前方块），重绘成本低
- 简化代码逻辑（无需跟踪每个元素的位置变化）
- 60fps 稳定

**布局**（左到右）：
```
[Hold 4×4] [Gap 16] [Main Grid 10×20] [Gap 16] [Next 4×4 × 3]
```

### 5.4 缓冲区设计

- **内部 grid** 22 行（含 2 行缓冲区）
- **外部显示** 20 行（不含缓冲区）
- 缓冲区用于方块从顶部"入场"，避免视觉上"瞬移"
- 锁定时写入缓冲区也合法（因为 `y >= 0`），但 `getDisplayGrid()` 切片后不显示

**为什么 2 行**：I 块最高 4 行，方块在 y=0 出生时上方还有 2 行隐藏空间。

### 5.5 SRS 严格按官方表

- J/L/S/T/Z 共用 KICKS_JLSTZ（5×4 = 20 偏移）
- I 单独 KICKS_I
- 每个旋转状态 5 个测试点（[0,0] 优先）

**逆时针**：复用顺时针表，反向索引
- CCW 0->L 等同 CW 0->R 的反向
- CCW R->0 等同 CW L->0 的反向

### 5.6 7-bag 随机

```typescript
class Bag {
  private current: TetrominoType[];
  
  take(n): TetrominoType[] {
    // current 不足时重新洗牌
    // Fisher-Yates shuffle
  }
}
```

- 每次洗牌打乱 7 种方块
- 保证每 7 个方块内 7 种都恰好出现 1 次
- 比纯随机更可预测（不出现"连续 5 个 I 块"）

### 5.7 主循环 60Hz 固定步长

```typescript
private readonly LOGIC_STEP = 1000 / 60;  // 16.67ms

private loop(timestamp: number): void {
  const delta = timestamp - this.lastTime;
  this.accumulator += delta;
  while (this.accumulator >= this.LOGIC_STEP) {
    this.update(this.LOGIC_STEP);
    this.accumulator -= this.LOGIC_STEP;
  }
  this.render();
  this.rafId = requestAnimationFrame((t) => this.loop(t));
}
```

**好处**：
- 物理模拟与帧率解耦
- 标签页隐藏后恢复不会出现"瞬间下落 N 格"
- delta clamp 100ms 防止长任务后暴跳

### 5.8 引擎 → UI 解耦

**核心思想**：引擎不 import 任何 UI 组件，仅通过 callbacks 通知。

```typescript
interface GameEngineCallbacks {
  onPhaseChange: (phase: GamePhase) => void;
  onStateChange: (snapshot: GameSnapshot) => void;
  onLinesClear: (count: number, isTetris: boolean) => void;
  onLevelUp: (level: number) => void;
  onGameOver: (score: number, isNewRecord: boolean) => void;
}
```

UI 层在 `TetrisGame.tsx` 实例化引擎时注入：

```typescript
const engine = new GameEngine({
  canvas,
  callbacks: {
    onPhaseChange: (p) => useGameStore.getState().setPhase(p),
    onStateChange: (snap) => { ... },
    // ...
  },
});
```

## 六、状态管理详解

### 6.1 引擎私有状态

```typescript
class GameEngine {
  // 不可变状态
  private phase: GamePhase = 'menu';
  private board: Board;
  private bag: Bag;
  private current: Tetromino | null;
  private nextQueue: TetrominoType[];
  private holdType: TetrominoType | null;
  private holdUsed: boolean;
  private score: number;
  private highScore: number;
  private level: number;
  private lines: number;
  private combo: number;
}
```

### 6.2 Zustand 全局状态

```typescript
interface GameStore {
  // ============ 状态 ============
  phase: GamePhase;
  score: number;
  highScore: number;     // ← 持久化
  level: number;
  lines: number;
  combo: number;
  holdType: TetrominoType | null;
  nextQueue: TetrominoType[];
  isNewRecord: boolean;
  audioEnabled: boolean; // ← 持久化
  flashLines: number | null;
  isTetris: boolean;
  levelUpLevel: number | null;
  
  // ============ actions ============
  setPhase / setScore / setLevel / ...
  toggleAudio / setAudioEnabled
  resetRound / clearFlash / clearLevelUp
}
```

**持久化**：
- `name: 'tetris:store'`
- `partialize`: 仅 `highScore` + `audioEnabled`

## 七、性能优化

### 7.1 渲染层

- 单次 `clear()` + 全量重绘（元素少，无须增量）
- 动画状态由 `setTimeout` 驱动（400ms / 1200ms 自动消失）
- 阴影 / 高光等视觉细节按需绘制

### 7.2 状态层

- 引擎不订阅 store，store 仅在事件触发时更新
- UI 组件用 selector 精确订阅（避免无关 re-render）
- `React.memo` 包裹纯展示组件

### 7.3 引擎层

- 60Hz 固定步长主循环，物理与渲染解耦
- `delta` 上限 100ms，防止长任务后暴跳
- 7-bag 队列预填（4 个 = 3 显示 + 1 缓冲），避免 spawn 时延迟
- Ghost / Hold / Next 复用同一渲染器（共享 4×4 矩阵绘制逻辑）

### 7.4 输入层

- 键盘事件 `preventDefault()` 阻止默认滚动
- 触屏 `passive: true` 提升滚动性能
- 不做 DAS（Delayed Auto Shift）— V2 阶段

## 九、模块依赖关系

```
TetrisGame.tsx
    │
    ├─► GameEngine ──► Board ──► Tetromino ──► SRS
    │      │            │           │
    │      │            │           └─► tetrominoes
    │      │
    │      ├─► Bag
    │      ├─► Renderer
    │      ├─► Input
    │      └─► AudioSystem
    │
    ├─► useGameStore ──► storage
    │
    └─► UI components (HUD/MainMenu/...)
            │
            └─► ui (button/card/badge/...)
```

**依赖方向**：
- UI → 引擎（单向）
- 引擎 → 工具（Audio / Storage / Config）
- 引擎 → 引擎内部（Board ↔ Tetromino）

**禁止**：
- 引擎 → UI（框架无关原则）
- 引擎 → Zustand（解耦）
- 状态层 → 引擎（避免循环依赖）
