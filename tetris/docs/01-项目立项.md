## 一、项目背景

### 1.1 立项动机

俄罗斯方块诞生于 1984 年，由 Alexey Pajitnov 创造，是史上最畅销的电子游戏（销量超 2 亿份）。它涉及方块旋转、碰撞检测、消行算法、随机生成（SRS 体系）等核心 2D 游戏概念，是真正意义上的"游戏开发"练手项目。

选择它作为本项目库（Web_Game_01）的第三款产品，原因有三：

1. **技术代表性**：Tetris 是网格类游戏的"集大成者"，SRS 旋转 + 7-bag 随机 + Hold/Next + Ghost + 速度曲线，几乎涵盖所有 2D 游戏算法。
2. **教学价值**：SRS 官方踢墙表是行业标准，掌握后可触类旁通到推箱子、连连看等网格游戏。
3. **复用性强**：项目骨架（React + Zustand + Canvas + Vitest）继承自 snake / 2048，是项目库"v2.0 质量优先栈"的第三款落地。

### 1.2 项目定位

- **类型**：2D 网页休闲游戏
- **风格**：经典 SRS 配色（青/黄/紫/绿/红/蓝/橙）+ 暗黑紫外框（项目库设计语言）
- **目标平台**：桌面浏览器（Chrome / Edge / Firefox / Safari）+ 移动端浏览器（触屏滑动）

## 二、项目目标

### 2.1 学习目标（核心）

| 序号 | 能力 | 衡量标准 |
| --- | --- | --- |
| 1 | SRS 旋转系统 | 严格按 tetris.wiki 官方踢墙表实现 + 单元测试覆盖 |
| 2 | 7-bag 随机生成 | Fisher-Yates 洗牌 + 队列管理 |
| 3 | 多区域 Canvas 渲染 | 主网格 + Hold + Next 一次性绘制 |
| 4 | 物理模拟 | 重力 + 软降/硬降 + Lock Delay |
| 5 | NES 经典计分 | 单消 100/双消 300/三消 500/Tetris 800，× 当前等级 |
| 6 | 等级系统 | 每 10 行 +1 级，速度按曲线递减 |
| 7 | 状态机 | menu / playing / paused / over 四阶段 |
| 8 | 触屏控制 | 滑动 + 点击双输入 |

### 2.2 产品目标

- ✅ 交付一款可在浏览器流畅运行的 Tetris 游戏
- ✅ 实现 PRD-03 中 MVP 全部 10 项功能（F-01 至 F-10）
- ✅ 完成至少 7 项 V2 扩展（F-11 Next / F-12 Hold / F-13 等级 / F-14 连击 / F-15 Ghost / F-16 暂停 / F-18 音效）
- ✅ 可通过 Vercel 一键部署

### 2.3 质量目标

| 指标 | 目标 |
| --- | --- |
| 性能 | 60fps，Level 20+ 无掉帧 |
| 兼容性 | Chrome 100+ / Edge 100+ / Firefox 100+ / Safari 15+ / 移动端主流浏览器 |
| 代码质量 | TypeScript 严格模式，ESLint 通过，核心算法单测全绿 |
| 体积 | 首屏加载 ≤ 500KB（gzip） |
| 测试覆盖 | 旋转/碰撞/消行/Hold/计分 100% 覆盖 |

## 三、范围定义

### 3.1 ✅ 包含范围

| 模块 | 范围 |
| --- | --- |
| 网格 | 10×20 + 2 行缓冲区 |
| 方块 | I / O / T / S / Z / J / L 共 7 种 |
| 旋转 | SRS 踢墙表（JLSTZ 共用 + I 单独） |
| Hold | 锁键：每方块 1 次 |
| Next 预览 | 前 3 个 |
| 阴影（Ghost） | 当前方块落地位置预览 |
| 计分 | NES 经典 × 等级 + 连击 |
| 等级 | 每 10 行 +1 级，下落速度递减 |
| 暂停 | P 键 / 暂停遮罩 |
| 音效 | Web Audio API 合成 9 个音效 |
| 视觉 | 经典 SRS 配色 + 暗黑紫边框 |
| 存储 | localStorage（最高分 + 音效开关） |

### 3.2 ❌ 不包含范围

| 不包含 | 原因 |
| --- | --- |
| 联机对战 | 单机休闲项目 |
| 自定义方块皮肤 | PRD 明确排除 |
| 撤销（Undo） | Tetris 不允许 |
| 录像回放 | 不在 V1 范围 |

> **V2+ 深度优化已实现**：T-Spin 计分、Lock Delay（500ms 触底后微调）、DAS/ARR（长按加速）、Perfect Clear、B2B Chain、PWA 离线支持均已落地，详见 [02 - 需求拆分](./02-需求拆分.md)。

## 五、风险评估

| 风险 | 概率 | 影响 | 应对措施 |
| --- | --- | --- | --- |
| SRS 踢墙表写错 | 中 | 高 | 严格按 tetris.wiki 官方表，单元测试逐 case 覆盖 |
| 高速下落输入丢失 | 中 | 中 | 60Hz 固定逻辑步长 + 输入事件去抖 |
| 移动端无键盘 | 中 | 中 | V2 阶段触屏滑动 + 滑入阈 30px |
| 60fps 渲染掉帧 | 低 | 中 | 一次性绘制整帧，避免 React re-render 干预渲染 |
| AudioContext 锁（iOS Safari） | 中 | 低 | 首次用户交互后 resume |
| localStorage 配额超限 | 低 | 低 | 静默吞掉异常 |

## 六、资源投入

### 6.1 人力

- **1 人全栈开发**（前端为主，可独立完成）

### 6.2 时间

- **总工期**：4-5 天（按每天 6-8 小时有效开发计）

### 6.3 技术资源

- Node.js 18+ / npm 9+
- Vite 5.x
- TypeScript 5.x 严格模式
- Zustand 4.x
- Framer Motion 11.x
- shadcn/ui（Radix UI + Tailwind CSS）

- happy-dom（仅测试用）
- Web Audio API（零素材依赖）
