| 样式 | Tailwind CSS | 3.4 |
| 组件库 | shadcn/ui（Radix UI + CVA） | latest |

## 二、核心选型详解

### 2.1 Vite 5.x

**理由**：
- ⚡ 开发启动 < 500ms，HMR < 50ms
- 🔧 零配置支持 TypeScript / JSX / CSS Modules
- 📦 产出 ES2020，浏览器兼容性好
- 🧪 与 Vitest 无缝集成（共享配置）

**对比**：
- ❌ Webpack 5：启动慢，配置复杂
- ❌ Parcel：生态弱，Vercel 集成不佳

### 2.2 React 18.3

**理由**：
- 🪝 Hooks + Context 足够（本项目用 Context 暴露 engine）
- 🔄 Concurrent Mode 留作未来扩展
- 📚 生态成熟，学习成本低

**对比**：
- ❌ Vue 3：项目库统一用 React
- ❌ Solid.js：生态弱

### 2.3 TypeScript 5.4 严格模式

**理由**：
- 🛡️ 严格模式（`strict: true`）捕获 90% 潜在 bug
- 🎯 泛型 + 联合类型 + 字面量类型适合 Tetris（TetrominoType / Rotation）
- 🧪 Zod 配合实现运行时校验

**配置要点**：
```json
{
  "compilerOptions": {
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "noImplicitReturns": true
  }
}
```

> **注**：`noUncheckedIndexedAccess` 未启用。该选项会让所有索引访问返回 `T | undefined`，对 Tetris 网格 / 矩阵访问（`grid[y][x]`、`shape[row][col]`）会引入大量非空断言与类型噪音，权衡后保持关闭，依赖边界检查 + 单元测试保证安全。

### 2.4 Tailwind CSS 3.4

**理由**：
- 🎨 原子化样式，构建时按需生成，体积小
- 🧩 配合 shadcn/ui（基于 Tailwind 的可复制组件）
- 🌑 暗黑模式天然支持（`class="dark"`）

**配置要点**：
- 自定义设计 token：背景 zinc-950、主色紫色 (hsl 263 70% 50%)、强调青色
- 字体：-apple-system / SF Mono
- 动画：fade-in / scale-in / flash

### 2.5 shadcn/ui

**理由**：
- 🧩 复制源码到自己项目，可定制性高
- ♿ Radix UI 提供无障碍支持
- 🎨 与 Tailwind 无缝集成

**复制组件清单**（自 2048）：
- `button.tsx`、`badge.tsx`、`dialog.tsx`、`switch.tsx`、`label.tsx`

> **注**：`card.tsx`、`separator.tsx` 已移除（本项目未使用）。

### 2.6 Zustand 4.5

**理由**：
- 🪶 1.5KB，比 Redux 轻 95%
- 🪝 Hooks 风格，TypeScript 友好
- 💾 persist 中间件支持 localStorage

**对比**：
- ❌ Redux Toolkit：模板代码多，本项目过度设计
- ❌ Jotai：项目库已统一 Zustand

**Tetris 用法**：
- `tetris:store`：持久化 highScore + audioEnabled
- 引擎 → UI：通过 callbacks 推送（不每帧推送）
- UI → 引擎：通过 Engine Context 暴露 engine 实例

### 2.7 Canvas 2D

**理由**：
- 🎮 60fps 实时渲染，多区域一次绘制
- 🖼️ Ghost / Hold / Next / 主网格 共享一个 canvas
- 🪶 比 SVG 轻，性能更优

**对比**：
- ❌ SVG / DOM：节点多，性能差
- ❌ WebGL：Tetris 2D 没必要

**关键设计**：
- 每帧 `clear()` + 全量重绘
- 动画状态由外部驱动（通过 setClearAnimation / setLevelUpAnimation）
- Buffer 行不渲染（Y < 0 跳过）

### 2.8 Vitest 1.6

**理由**：
- 🧪 Vite 原生测试框架，零配置
- ⚡ 比 Jest 快 2-5 倍
- 🪝 与 Vite 共享配置（alias、transform）

**Tetris 测试**：
- 6 个测试文件：Board / Tetromino / Bag / GameEngine / ScoringSystem / LockDelayManager
- 148 个测试用例
- happy-dom 用于 GameEngine（需要 Canvas + AudioContext）

### 2.9 happy-dom 14

**理由**：
- 🪶 比 jsdom 轻 70%（约 1MB vs 7MB）
- ⚡ 比 jsdom 快 3-5 倍
- 🧪 适合测试需要少量 DOM API 的代码

**Tetris 用法**：
- 仅 `GameEngine.test.ts` 使用 `// @vitest-environment happy-dom`
- 其他测试（Board / Tetromino / Bag）保持默认 `node` 环境
- Canvas getContext 需手动 stub（happy-dom 不支持）

### 2.10 Framer Motion 11

**理由**：
- 🎬 声明式动画，AnimatePresence 自动处理组件退出
- 🎯 用于菜单淡入、数字弹动、遮罩切换

**Tetris 用法**：
- `MainMenu` / `PauseOverlay` / `GameOverModal` 切换动画
- `HUD` 数字变化 spring 弹动
- `Badge` 破纪录缩放入场

### 2.11 Web Audio API（原生）

**理由**：
- 🆓 零素材依赖
- 🪶 代码量小（约 250 行 9 个音效）
- 🔧 完全可控

**Tetris 音效清单**（9 个）：
- `playMove` 短 click 800→400Hz / 50ms
- `playRotate` 中频 blip 600Hz / 60ms
- `playLock` 低 thud 200→80Hz / 80ms
- `playClear(n)` 按 n 变化（1/2/3/Tetris 三音和弦）
- `playDrop` 下行扫频 500→100Hz / 100ms
- `playHold` 双音 swap 500/700Hz
- `playLevelUp` 上行三音 C5-E5-G5
- `playGameOver` 下行音 440→110Hz / 400ms
- `playClick` 暂停/继续 600Hz / 40ms

### 2.12 Zod 3.23

**理由**：
- 🛡️ 运行时类型校验
- 🧩 配合 TypeScript 实现"静态 + 动态"双保险
- 📦 体积小（10KB）

**Tetris 用法**：
- `gameConfigSchema`：校验 CONFIG 对象

### 2.13 部署：Vercel

**理由**：
- 🚀 零配置 Vite 部署
- 🔄 自动 Git 集成
- 🌍 全球 CDN
- 💚 免费层足够个人项目

**配置**：
- `vercel.json`：SPA 重写规则（所有路径 → index.html）
- base: `./`（相对路径，适配任意子路径）

## 三、放弃的方案

| 方案 | 放弃原因 |
| --- | --- |
| Cocos Creator / Phaser | 重量级游戏引擎，Tetris 2D 不需要 |
| Pixi.js | WebGL 渲染，2D Tetris 过度 |
| HTML5 Canvas + React-Three-Fiber | 3D 化，无意义 |
| Pinia（Vue） | 项目库统一 React |
| 阿里云 / 腾讯云 | 备案 + 收费，个人项目用 Vercel |
| Tailwind v4 | 当时未稳定，继续用 3.4 |
| React Native | 移动端通过 Web 触屏解决，不做 native |

## 四、技术决策记录

| 决策 | 备选 | 选定 | 理由 |
| --- | --- | --- | --- |
| 渲染 | Canvas / SVG / WebGL | Canvas 2D | 性能 + 复杂度平衡 |
| 状态管理 | Redux / Zustand / Jotai | Zustand | 轻量 + 简单 |
| 测试环境 | node / jsdom / happy-dom | node + happy-dom | 99% 测试 node，仅 GameEngine 用 happy-dom |
| 动画 | CSS / Framer Motion / GSAP | Framer Motion | 与项目库统一 |
| 部署 | Vercel / Netlify / Cloudflare | Vercel | 集成度最高 |
| TypeScript 严格度 | strict / noImplicitAny / noUncheckedIndexedAccess | strict + noImplicitAny + strictNullChecks（不启用 noUncheckedIndexedAccess） | 网格访问频繁，避免 undefined 噪音 |
| 7-bag 随机 | Math.random / Seeded | Math.random（可注入 RNG） | 默认随机，可测试 |
| Hold 锁 | 锁定到游戏结束 / 每方块 1 次 | 每方块 1 次 | Tetris Guideline 标准 |
